# CS 300: Data Structures and Algorithms

## Repository Overview

This repository contains projects completed for CS 300: Data Structures and Algorithms at Southern New Hampshire University. The projects focus on algorithm analysis, runtime evaluation, and the implementation of data structures in C++. Included are analyses of vector, hash table, and binary search tree data structures, as well as a course advising application designed to organize and retrieve academic course information efficiently.

## Reflection

### What was the problem you were solving in the projects for this course?

The primary problem addressed throughout this course was determining how to efficiently store, organize, retrieve, and sort data. The projects focused on helping academic advisors access course information by loading course data from a file, displaying courses in alphanumeric order, and retrieving detailed information about specific courses and their prerequisites. Solving this problem required evaluating different data structures and selecting the most appropriate solution based on performance and functionality requirements.

### How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by analyzing the strengths and weaknesses of multiple data structures, including vectors, hash tables, and binary search trees. Understanding data structures is important because the choice of structure directly affects the efficiency of searching, sorting, inserting, and storing data. By comparing runtime and memory considerations, I was able to evaluate which structure best met the application's requirements and implement solutions that balanced performance with maintainability.

### How did you overcome any roadblocks you encountered while going through the activities or project?

Several challenges arose while implementing sorting algorithms, hash tables, and binary search trees. Debugging compile errors, handling file input correctly, and ensuring that data was stored and displayed accurately required careful testing and revision. I overcame these obstacles by breaking problems into smaller components, verifying each section of code independently, and using program output to validate that the algorithms behaved as expected.

### How has your work on this project expanded your approach to designing software and developing programs?

This course reinforced the importance of planning before coding. Developing pseudocode, evaluating multiple design options, and analyzing algorithm performance demonstrated that software design decisions have a significant impact on the final product. I now place greater emphasis on selecting appropriate data structures early in development rather than focusing only on implementing functionality.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

The projects encouraged me to write code using modular functions with clearly defined responsibilities. Separating tasks such as file loading, sorting, searching, and displaying information made the programs easier to understand and modify. I also gained a greater appreciation for meaningful variable names, consistent formatting, and documentation, all of which improve readability and make future updates easier to implement.
